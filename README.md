# mileage-expense-tracker-893

Full-stack mileage and expense tracker — AWS backend + iOS app.  
Tracks vehicle trips using **both odometer readings and GPS simultaneously**, logs expenses with **receipt scanning via AWS Textract**, and stores everything in DynamoDB.

## iOS App — MileageTracker893

**Bundle ID:** `com.dliv.MileageTracker893`  
**Min iOS:** 17+  
**Auth:** Plain URLSession Cognito (no Amplify SDK)  

### Key design decisions
- **Zero hardcoded strings** — all backend config loaded from `met_outputs.json` at startup via `AppConfig.swift`
- **No Amplify SDK** — plain `URLSession` + Keychain for auth. Swap backends by updating `met_outputs.json` only
- **JWT auto-refresh** — `AuthService` silently refreshes expired tokens via Cognito `REFRESH_TOKEN_AUTH`

## Trip Tracking — Dual Measurement

Every trip records **both** GPS and odometer distance simultaneously.

**Flow:**
1. Select vehicle → enter **start odometer** → tap **Start**
2. GPS tracks live distance while driving (big real-time display)
3. Tap **End Trip** → enter **end odometer**
4. App shows a comparison of both measurements before saving

**What gets stored in DynamoDB:**
```
startOdometer     ← entered at trip start
endOdometer       ← entered at trip end
odometerDistance  ← endOdometer - startOdometer
gpsDistance       ← CoreLocation measured miles
tripDate          ← YYYY-MM-DD
purpose           ← optional
```

The trip list shows both measurements side by side:
```
Odometer: 44.0 mi  ·  GPS: 43.8 mi
```

## Expense & Receipt Tracking

**Categories:** Fuel, Maintenance, Insurance, Parking, Tolls, Other

**Receipt OCR flow:**
```
1. Add expense → attach photo from Photos library
2. App creates expense record → gets expenseId back from API
3. Receipt uploaded to S3 at: receipts/{expenseId}.jpg
4. S3 event fires OCR Lambda automatically
5. Textract extracts: total, date, merchant, line items
6. Results written back to expense record in DynamoDB
7. App shows "Receipt scanned ✓" with extracted data
```

## Auth — Shared Cognito Pool

The mileage app shares dliv.com's Cognito User Pool with its own App Client.

**Phase 1 (current):** All dliv.com users can access both apps.  
**Phase 2 (later):** Add Pre-Token Generation Lambda to dliv.com's App Client to enforce `dliv-access` group membership. Mileage-only users go in `mileage-access` group.

## API Reference

All endpoints require `Authorization: {CognitoIdToken}` header.

| Method | Path | Description |
|---|---|---|
| GET | /vehicles | List all vehicles |
| POST | /vehicles | Create vehicle |
| PUT | /vehicles/{vehicleId} | Update vehicle |
| DELETE | /vehicles/{vehicleId} | Delete vehicle |
| GET | /trips | List all trips |
| GET | /trips?vehicleId=xxx | Filter by vehicle |
| POST | /trips | Create trip |
| PUT | /trips/{tripId} | Update trip |
| DELETE | /trips/{tripId} | Delete trip |
| GET | /expenses | List all expenses |
| GET | /expenses?vehicleId=xxx | Filter by vehicle |
| POST | /expenses | Create expense |
| PUT | /expenses/{expenseId} | Update / attach receipt |
| DELETE | /expenses/{expenseId} | Delete expense |
